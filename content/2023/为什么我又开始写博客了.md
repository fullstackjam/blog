过去一年多，我停更了。脑子里不是没想法，但就是提不起劲写下来。现在回头看，挺可惜的。

<!--more-->

## 我的博客黑历史

说起来，我写博客这件事可以追溯到 2016 年大学一年级。那时候用一键 LNMP 搭了个 WordPress，注册了个便宜域名，感觉自己拥有了全世界。

之后就开始了漫长的折腾之旅：WordPress → Typecho → Hexo，每换一次框架都觉得「这次一定能坚持写下去」。结果每次都是折腾完框架就没了动力，文章没写几篇，博客倒是搭了好几个。

更惨的是，那些早期文章全丢了。因为完美主义作祟——总觉得自己写的东西不够好，别人肯定写过更好的版本。于是要么删掉，要么存在草稿箱里永远不发。现在想想，哪怕是很稚嫩的文字，留下来也是一种记录啊。

## 极简主义的启发

转折点是看到 [eaimty](https://github.com/EAimTY) 的[这篇博客](https://www.eaimty.com/2023/recent-updates-and-new-blog/)。他的网站干净到极致，没有花哨的动画，没有复杂的布局，就是内容本身。

这让我突然想明白了一件事：**博客最值钱的部分是内容，不是皮肤**。

于是我选了 [Zola](https://www.getzola.org/)。原因很简单：

- **单个二进制文件**，不依赖 Node.js 或 Go，重装系统后秒恢复
- **模板语法直观**，Tera 比 Go template 好写太多
- **够简单**，不会让我又陷入「折腾框架」的循环

整个网站除了评论系统，没有一行 JavaScript，也没有外部 CSS。评论用 GitHub Issues 做后端，前端通过我在 Cloudflare Worker 上部署的代理来调用 GitHub API（主要是为了解决 CORS 问题和简化认证流程）。所有评论数据都存在 GitHub Issue 里，透明、可迁移。

这次我告诉自己：**别折腾框架了，写就对了**。

## 想写什么

技术方面的东西为主。折腾软硬件、搭服务、踩坑记录，什么都行。

技术成长这种事不是一朝一夕的，我希望多年以后翻回来看，能看到自己一路走过来的痕迹。哪怕写得不好，至少是真实的。

## 聊聊近况

过去这一年变化挺大的。我从一家小互联网公司跳到了一家跨国金融公司，工作内容和生活方式都变了。

最大的变化是——我开始写代码了。之前大半年一直在做开发工作，对我来说算是全新的领域。很难说在哪家公司过得更好，但坦白讲，精神状态一直在下滑，对工作和学习的热情也在慢慢消退。眼看就要工作三年了，技术上的成长好像没达到自己当初的预期。

不过焦虑归焦虑，我开始学着接受自己就是个普通人。与其什么都想要，不如选一个方向持续投入。对自己的爱好花点精力和钱，找到生活的乐趣，可能比追着 KPI 跑更重要。

## 新玩具和新计划

就在写这篇文章的时候，我入手了 2022 款 M2 Pro MacBook Pro（32G+1T）。旧 PC 也没闲着，直接改造成了 Homelab。

这不只是硬件的重新利用，也是我认真学技术的一个新起点。接下来打算好好研究 Kubernetes 和 CI/CD，算是给自己的职业发展做点储备。

后面几篇文章会分享搭建 Homelab 的经验——硬件选择、域名备案、服务部署这些。希望能帮到也想折腾 homelab 的朋友，同时也给自己留个记录。

## 最后

信息太多了，我越来越觉得需要找到属于自己的节奏，别被每天的信息流冲着走。

希望未来能找到我想要的生活方式，也希望这个博客能把这段路一步步记下来。
